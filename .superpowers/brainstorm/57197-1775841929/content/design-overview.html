Animation Design — Component Breakdown

All CSS-based, no new dependencies. One shared keyframe block added to index.css.

Shared keyframes added to index.css
fadeUp opacity 0→1 + translateY 12px→0, used for page-load stagger
barGrow scaleY 0→1 from bottom, used for chart bars
shimmer moving gradient, used for skeleton loading
All durations 200–500ms, easing ease-out or cubic-bezier(0.34,1.56,0.64,1) for bars.
Component animations
Home.tsx
Page load — staggered fade-up
Left column cards animate in at 0ms, 80ms, 160ms. Right column at 80ms, 160ms. Gives a gentle cascade on every route change.
animate-fadeUp animation-delay inline
AgentCard.tsx
Skeleton while loading
Replace the "LOADING..." text with shimmering placeholder bars for the name and role lines. Existing loading state already controls this.
animate-shimmer bg-[#E8E4DB] rounded
ResolutionChart.tsx
Bars grow in on mount
Each bar animates from scaleY(0) to its full height on mount. Bars stagger 60ms apart left-to-right.
animate-barGrow transform-origin: bottom animation-delay inline
ChatModal.tsx · TestChatModal.tsx
Fade + scale on open/close
Backdrop fades in (opacity 0→0.4). Panel scales 0.97→1 and fades in together. CSS transition driven by an isVisible state that lags one tick behind open.
transition-opacity transition-transform scale-97 → scale-100
Sidebar.tsx · TopBar.tsx
Active state transition
Both already have transition-colors. Adding duration-150 makes the active background pill animate smoothly instead of snapping.
duration-150 ease-in-out
Login.tsx · Signup.tsx · Buttons
Press feedback
All primary buttons get active:scale-[0.97] so they feel tactile on click. One class, added to each button.
active:scale-[0.97] transition-transform duration-100
